## 死循环 / 大循环：已有防护（甚至可以当亮点演示）

**实测（2026-06-08）：`while True`、`range(10**8)` 不会再冻死标签页了。** 两层防护：

1. **运行前拦截**：点"运行"前先静态检查，`while True` / `while 1` / `range(10**8)` 这类会被**当场拦下**，终端提示"Nexus 预检拦截：检测到明显无退出条件的循环"，小助手还会顺势讲解。
2. **6 秒硬超时**：浏览器内 Python 跑在独立 worker 线程，万一有漏网的，最多 6 秒被强制终止——主线程（标签页）全程不卡。

**所以这反而可以当卖点演示**：故意写一句 `while True: pass` → 点运行 → 展示它被**拦截 + 导师讲解为什么**。比"小心翼翼不敢碰"更有说服力（已线上实测：拦截生效、标签页不卡）。

> 仍建议：常规教学就跑小代码（如 `for i in range(1, 6)`）。`input()` 在沙箱里等不到输入、最多 6 秒超时停止——不是卡死，但没意义，别拿它演示。

## ✅ 安全的演示动线（3–4 分钟）

1. 打开演示地址，进入 Lv.1（Python · 初级）。
2. 直接点「**运行**」跑出预置的 `print(...)` —— 展示浏览器内即时执行 Python。
3. 改一行，演示一个**小**循环（安全示例）：
   ```python
   for i in range(1, 6):
       print(i)
   ```
   （范围小、瞬间结束，绝不要把 5 改成很大的数。）
4. 展示左侧**小助手引导面板**：
   - 「这关真正要学」「实践路线」「错题本·间隔复盘」。
   - **「你想做个什么？」定制路线**：未登录时这里展示的是**示例预览**（猜数字游戏那张虚线卡片）。可以讲解理念，但**不要点"生成"**——真正的生成需要登录，演示不登录。
5. 右侧 AI 小助手：可问一句"`for` 循环是怎么数数的？"展示讲解（小助手回复需要后端，已验证可达）。

## 本地运行（仅备用，现场优先用线上）

如果一定要本地跑：

```bash
npm run build && npm start
```

**不要用 `npm run dev`**——dev 模式慢、会常驻进程、不代表生产行为。
`npm start` 前必须先 `npm run build`。仓库根目录在 `编程工坊/frontend/`。
